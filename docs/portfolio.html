H&M 대규모 데이터 기반 구매 예측(Next-Item Prediction) 및 고성능 커머스 시스템

Service ecommerce.p-e.kr
Repository E-commerce Systems
Duration 2025.11 - 2026.03

H&M Group의 대규모 의류 데이터와 실제 구매 트랜잭션 로그를 분석하여, 마르코프 체인(Markov Chain) 기반의 '차순위 구매 예측(Next-Item Prediction)' 엔진을 구현함. 대규모 트래픽 환경에서의 실시간 추천을 위해 CQRS 패턴과 Redis 캐싱 레이어를 도입하여 DB 부하를 분산하고 응답 지연을 최소화함. k6 부하 테스트를 통해 검색 Latency 8배 단축, 인증 에러율 0% 등 정량적인 성능 지표를 확보함.

성과 지표 (Performance Metrics)

Performance CQRS 도입을 통한 검색 Latency 최적화

다중 조인 기반의 DB 병목을 해결하기 위해 읽기 전용 역정규화 테이블을 구축함. Zero-Join 설계를 통해 검색 Latency를 기존 대비 8.02배 단축하며 시스템 응답성을 확보함.

Scalability Redis 캐싱을 이용한 고가용성 추천 엔진

Redis 캐시 레이어 도입 및 단일 요청(Single-Request) 구조 최적화를 통해 200 VU 부하 테스트 시 CPU 점유율을 17.5%에서 12.0%로 개선하고 초당 6,852건(RPS)을 처리함.

Stability Stateless 인증 체계 및 결함률 0% 달성

고부하 시 RDB의 Race Condition으로 발생하던 인증 실패(1.58%) 해결을 위해 Redis 기반 세션 저장소를 도입함. 100 VU 고강도 테스트 결과, 에러율 0.00%의 결함 제로 상태를 기록함.

AI/ML 마르코프 체인 기반 구매 예측 정밀도 향상

사용자 구매 여정을 상태 전이 확률로 모델링하여 MAP@20 기준 기존 인기순 모델 대비 10배 이상의 정확도(0.1052)를 기록하며 개인화 추천의 유효성을 확인함.

Ops Docker Swarm 기반 무중단 운영 환경

GitHub Actions와 Docker Swarm을 연동한 CI/CD 파이프라인을 구축하여 자동화된 배포 체계를 마련함. Prometheus/Grafana를 통해 시스템 메트릭을 실시간 모니터링하며 가시성을 확보함.

기술적 난관과 문제 해결

01. CQRS 및 역정규화 기반의 검색 성능 최적화

  • 문제 정의 및 배경: 서비스 규모 확장에 따라 상품 조회 시 10여 개의 테이블을 조인하는 Multi-Join 부하가 발생했으며, 응답 시간이 100ms를 상회하며 DB CPU 점유율이 급증함.
  • 기술적 해결 방안: 쓰기(Command) 모델과 읽기(Query) 모델을 분리하는 CQRS 패턴을 도입함. 검색 전용 역정규화 테이블(ProductSearch)을 구축하여 Zero-Join을 달성하고, 복합 인덱스를 최적화함.

02. Redis Caching을 통한 API 응답 속도 및 자원 효율화

  • 문제 정의 및 배경: 추천 엔진 등 연산 집약적 로직이 매 요청마다 DB I/O를 유발하여 시스템 전체의 병목 지점이 됨. 특히 반복적인 조회 요청은 불필요한 컴퓨팅 자원을 낭비함.

03. Stateless 아키텍처 구축을 통한 인증 안정성 및 확장성 확보

  • 문제 정의 및 배경: RDB 기반 인증 방식이 고부하 상황에서 물리적 I/O 지연으로 인한 Race Condition을 유발하며 1.58%의 에러율을 기록함. 또한 서버 확장 시 세션 동기화 한계에 직면함.
  • 기술적 해결 방안: 토큰 저장소를 Redis In-Memory로 이전하여 I/O 병목을 제거하고, JWT를 결합한 Stateless 인증 구조를 설계함. 모든 인스턴스가 단일 세션 저장소를 공유하여 수평 확장이 용이해짐.

04. 마르코프 체인 기반의 실시간 구매 예측 엔진 구현

  • 문제 정의 및 배경: 단순 인기순 추천 방식은 개별 사용자의 행동 맥락을 반영하지 못해 전환율 향상에 한계가 있음. 사용자의 다음 구매 행동을 예측할 수 있는 정교한 모델이 요구됨.
  • 기술적 해결 방안: 대량의 사용자 구매 여정을 분석하여 마르코프 체인 전이 확률 모델을 구축함. 현재 상태(최근 구매)를 기반으로 다음 상태(구매 확률이 가장 높은 상품)를 예측하는 엔진을 개발함.
    • Pros사용자 맞춤형 쇼핑 경험 제공을 통해 구매 전환율(CVR) 상승에 직접 기여
    Cons대규모 로그 데이터의 전처리 및 확률 행렬 연산을 위한 시스템 리소스 소모
