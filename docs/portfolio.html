H&M 대규모 데이터 기반 연관 구매 예측 및 고성능 커머스 시스템

서비스 URL ecommerce.p-e.kr
깃허브 URL E-commerce Systems
개발 기간 2025.11 ~ 2026.03

H&M Group의 1만 4천 건의 상품 데이터와 8만 건의 실제 구매 트랜잭션 로그를 분석하여, 현재 조회 중인 상품과 함께 구매될 확률이 높은 상품을 실시간으로 제안하는 연관 구매 예측(Next-Item Prediction) 엔진을 구현함. 대규모 트래픽 환경에서의 지연 없는 추천을 위해 CQRS 패턴과 Redis 캐싱 레이어를 도입하여 DB 부하를 분산하고 응답 속도를 극대화함. k6 부하 테스트를 통해 중앙값 응답 속도 20% 개선, 특정 테스트 시나리오 내 에러율 0% 등 정량적인 성능 지표를 확보함.

성과 지표 (Performance Metrics)

Performance CQRS 도입을 통한 검색 Latency 최적화

다중 조인 기반의 DB 병목을 해결하기 위해 읽기 전용 역정규화 테이블을 구축함. Zero-Join 설계를 통해 검색 Latency의 중앙값을 기존 대비 20% 단축하며 시스템 응답성을 확보함.

Performance Redis 캐싱을 이용한 고가용성 추천 엔진

Redis 캐싱 도입을 통해 200 VU 부하 테스트 시 CPU 점유율을 16.3%에서 12.1%로 개선하고 초당 6,800건(RPS) 이상의 요청을 안정적으로 처리함.

Performance Stateless 인증 체계 및 신뢰성 확보

고부하 시 RDB의 Race Condition으로 발생하던 인증 실패(1.58%) 해결을 위해 Redis 기반 세션 저장소를 도입함. 100 VU 고강도 테스트 결과, 에러율 0.00%의 안정적인 인증 프로세스를 검증함.

AI/ML 마르코프 체인 기반 연관 구매 예측 정밀도 향상

사용자 구매 여정을 상태 전이 확률로 모델링하여 현재 상품과 연관된 다음 구매 상품의 예측 정확도를 MAP@20 기준 인기순 모델 대비 10배 이상(0.1052) 향상시킴.

Ops Docker Swarm 기반 무중단 운영 환경

GitHub Actions와 Docker Swarm을 연동한 CI/CD 파이프라인을 구축하여 자동화된 배포 체계를 마련함. Prometheus/Grafana를 통해 시스템 메트릭을 실시간 모니터링하며 가시성을 확보함.

기술적 난관과 문제 해결

01. CQRS 및 역정규화 기반의 검색 성능 최적화

  • 문제 정의 및 배경: 서비스 규모 확장에 따라 상품 조회 시 10여 개의 테이블을 조인하는 Multi-Join 부하가 발생했으며, 응답 시간이 100ms를 상회하며 DB CPU 점유율이 급증함.
  • 기술적 해결 방안: 쓰기(Command) 모델과 읽기(Query) 모델을 분리하는 CQRS 패턴을 도입함. 검색 전용 역정규화 테이블(ProductSearch)을 구축하여 Zero-Join을 달성하고, 복합 인덱스를 최적화함.
  • V1 vs V2 성능 개선 핵심: 기존 V1은 ID만 반환하여 프런트엔드에서 추가 정보 조회를 위한 API 재호출(N+1 Network Call)을 유발했으나, V2는 역정규화 테이블에서 모든 메타데이터를 한 번에 서빙하여 네트워크 라운드트립을 최소화함.

02. Redis Caching을 통한 API 응답 속도 및 자원 효율화

  • 문제 정의 및 배경: 추천 엔진 등 연산 집약적 로직이 매 요청마다 DB I/O를 유발하여 시스템 전체의 병목 지점이 됨. 특히 반복적인 조회 요청은 불필요한 컴퓨팅 자원을 낭비함.
  • 기술적 해결 방안: Cache-Aside 전략 기반의 Redis 캐싱 레이어를 구축함. 추천 결과와 같은 준실시간성 데이터를 메모리에 상주시켜 DB 접근을 최소화하고 응답 속도를 극대화함.

03. Stateless 아키텍처 구축을 통한 인증 안정성 및 확장성 확보

  • 문제 정의 및 배경: RDB 기반 인증 방식이 고부하 상황에서 물리적 I/O 지연으로 인한 Race Condition을 유발하며 1.58%의 에러율을 기록함. 또한 서버 확장 시 세션 동기화 한계에 직면함.
  • 기술적 해결 방안: 토큰 저장소를 Redis In-Memory로 이전하여 I/O 병목을 제거하고, JWT를 결합한 Stateless 인증 구조를 설계함. 모든 인스턴스가 단일 세션 저장소를 공유하여 수평 확장이 용이해짐.

04. 실시간 연관 구매 예측(Frequently Bought Together) 엔진 구현

  • 문제 정의 및 배경: 단순 인기순 추천 방식은 사용자의 현재 관심사를 반영하지 못해 구매 전환율 향상에 한계가 있음. 현재 보고 있는 상품과 함께 구매될 확률이 높은 상품을 정교하게 제안할 모델이 요구됨.
  • 기술적 해결 방안: 고객의 구매 여정을 상태 전이 과정으로 모델링하는 마르코프 체인(Markov Chain) 엔진을 직접 구현함.
    • 구매 시퀀스 분석: 트랜잭션 로그를 시간순으로 정렬하여 '상품 A → 상품 B'로 이어지는 고객별 구매 여정을 상태 전이 데이터로 변환
    • 전이 확률 행렬 구축: 특정 상품 구매 후 다음 상품이 나타난 빈도를 확률화하여 전이 확률 행렬(Transition Matrix) 구축
    • 데이터 최적화(Top-20): 실시간 연산 부하를 제거하기 위해 전이 확률이 가장 높은 상위 20개의 연관 상품만을 선별하여 데이터 경량화
    • 고성능 서빙 통합: Python으로 사전 계산된 전이 확률을 메인 DB(CoPurchase)에 영속화하고, API 서버에서는 Cache-Aside 전략을 기반으로 Redis에 캐싱하여 추천 결과를 서빙함
    • Pros사용자의 현재 맥락(State)을 반영한 맞춤형 제안으로 실질적인 구매 전환율 상승 기여
    Cons대규모 로그 데이터의 전처리 및 확률 행렬 연산을 위한 배치 프로세스 필요
    Decision데이터의 영속성을 위해 RDB에 먼저 저장하되, 실시간 API 응답 성능을 보장하기 위해 Redis 캐싱 레이어를 결합한 하이브리드 저장 구조를 채택함.
    결과: 실제 트랜잭션 로그 기반 평가에서 인기순 모델 대비 10배 이상의 정확도 향상을 확인함.
    • MAP@20 (Mean Average Precision): 0.0098 → 0.1052 (10.7배 향상)
    • NDCG@20 (Ranking Quality): 0.0153 → 0.1235 (8배 향상)
